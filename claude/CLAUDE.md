# Self-Improvement Loop

## Session Start
- Always read:
  - NOTES.md
  - RULES.md
- Only read STRUCTURE.md if explicitly told that project structure changed.

## During Work
- Do not update any of these files mid-task.
- Track internally:
  - repeated mistakes
  - token-heavy patterns
  - unnecessary verbosity
  - useful shortcuts discovered

## Trigger Condition
- If prolonged token usage is detected (long session or multiple iterations), run self-improvement.

## Self-Improvement Actions

### 1. Update NOTES.md
- Delete all existing content.
- Replace with a minimal summary of the previous session:
  - what was done
  - key decisions
  - important context
- Keep it as short as possible without losing continuity.

### 2. Update RULES.md
- Append only.
- Add rules that:
  - reduce token usage
  - improve clarity
  - prevent repeated mistakes
- Do not rewrite existing rules.

### 3. Update STRUCTURE.md
- Only if project structure changed.
- If no structural changes:
  - do not read
  - do not modify

## Constraints
- Be concise. No redundancy.
- No explanations during updates.
- Do not perform updates unless trigger condition is met.
